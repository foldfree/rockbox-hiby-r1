# Winamp Classic theme ‚Äî HiBy R1

A pixel-faithful classic Winamp skin for Rockbox on the HiBy R1 (480√ó800
touchscreen, portrait). Recreates the Winamp main window, playlist window and
album-art window as a now-playing screen (WPS), with a matching base skin (SBS)
for the menus.

## What's here

```
winamp.cfg              theme config        -> /.rockbox/themes/winamp.cfg
wps/winamp.wps          now-playing screen  -> /.rockbox/wps/winamp.wps
wps/winamp.sbs          base skin (menus)   -> /.rockbox/wps/winamp.sbs
wps/winamp/*.bmp        skin bitmaps        -> /.rockbox/wps/winamp/
fonts/winamp_lcd.fnt    LCD digits          -> /.rockbox/fonts/
fonts/winamp_text.fnt   5√ó6 pixel text      -> /.rockbox/fonts/
tools/                  asset build scripts (see below)
```

Bitmaps (`wps/winamp/`):

- `backdrop.bmp` ‚Äî main WPS backdrop: WINAMP window + WINAMP PLAYLIST window +
  ALBUM ART window. Generated by `tools/build_backdrop.sh` from
  `backdrop.base.bmp` (the hand-tuned main-window body); the script overlays the
  titlebar header and draws the two lower windows.
- `backdrop.base.bmp` ‚Äî source body for the main window only (edit this, then
  re-run the build script).
- `sbsbackdrop.bmp` ‚Äî menu/base-skin backdrop (one framed window with a Winamp
  titlebar). Hand-finished ‚Äî see note in tools below.
- `analyzer.bmp` ‚Äî spectrum-bar fill, VISCOLOR green‚Üíred gradient.
- `posbar_groove.bmp` / `posbar_thumb.bmp` ‚Äî position-bar groove and slider thumb.

## Deploy

Copy each file to the path shown above under the device's `.rockbox`, then pick
**Winamp Classic** from Settings ‚Üí Theme Settings ‚Üí Browse Themes (or apply
`winamp.cfg`). Over adb:

```
D=/data/mnt/sd_0/.rockbox
adb push winamp.cfg            $D/themes/winamp.cfg
adb push wps/winamp.wps        $D/wps/winamp.wps
adb push wps/winamp.sbs        $D/wps/winamp.sbs
adb push wps/winamp/.          $D/wps/winamp/
adb push fonts/winamp_lcd.fnt  $D/fonts/
adb push fonts/winamp_text.fnt $D/fonts/
```

### Fonts

`winamp_lcd.fnt` and `winamp_text.fnt` are bundled. The theme also references two
**stock Rockbox fonts** that ship in the fonts package and are already on the
device ‚Äî they are not duplicated here:

- `22-Terminus-Bold.fnt` (kbps / kHz readouts)
- `14-Adobe-Helvetica-Bold.fnt` (UI font)

## Rebuilding the assets (`tools/`)

Requirements: ImageMagick (`magick`), Python 3, and Rockbox's `convbdf` (for the
fonts). The scripts also need the original **Winamp "Classic" skin BMPs**
(`TITLEBAR.BMP`, `NUMBERS.BMP`, `TEXT.BMP`, `POSBAR.BMP`, `VISCOLOR.TXT`, ‚Ä¶),
which are **not** committed here (third-party assets).

- `build_backdrop.sh` ‚Äî main WPS backdrop from `backdrop.base.bmp` + Winamp
  titlebar slices.
- `build_sbs.sh` ‚Äî menu backdrop (`sbsbackdrop.bmp`). Note: the committed
  `sbsbackdrop.bmp` was hand-finished after this script ran; re-running it
  overwrites those tweaks.
- `build_lcd_font.py` / `build_text_font.py` ‚Äî parse `NUMBERS.BMP` / `TEXT.BMP`
  into BDF, then `convbdf` ‚Üí `.fnt`.
- `grad.txt` ‚Äî VISCOLOR gradient used to paint `analyzer.bmp`.

The scripts were driven from a scratch working dir and still reference `/tmp`
paths (`SKIN=/tmp/winamp/skin`, output under `/tmp/winamptheme/...`); adjust the
path variables at the top of each before running. They are kept as a faithful
record of how the bitmaps were produced ‚Äî the committed assets above are the
authoritative, ready-to-deploy versions.

## All bitmaps must be uncompressed 24-bit BMP3

Rockbox silently rejects RLE/compressed BMPs and falls back to the default
theme. Always write skin bitmaps as:

```
magick in.png -type truecolor -compress none BMP3:out.bmp
```
